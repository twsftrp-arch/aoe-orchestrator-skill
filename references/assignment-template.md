# Assignment Template

오케스트레이터가 워커에 임무를 주입할 때(`aoe acp prompt <ID> '<메시지>'`) 쓰는 템플릿.
슬라이스는 **한 컨텍스트 안에 끝나는 크기**로 쪼갠다 — 전체 코드베이스급 분석·대규모 리팩토링은 장맥락 HQ로 라우팅.

## 임무 주입 템플릿

```
[프로젝트] <repo / 경로>
[목표]   <이 슬라이스에서 끝낼 한 가지. 한 컨텍스트 내 완결>
[맥락]   <배경·관련 파일·직전 결정. 길면 "먼저 docs/SESSION-HANDOFF.md를 읽어라">
[제약]
  - 슬라이스 시작 시 계획을 파일로 남겨라.
  - 체크포인트마다 docs/SESSION-HANDOFF.md를 fresh 세션이 이어받을 수준으로 갱신하라.
  - 컨텍스트는 캐시고 진실은 파일이다.
  - 권한: <commit/push 위임 여부 — local-profile / worker-contract 참조>
  - 위험 영역(스키마·인증·결제·배포·공개)은 손대기 전 STATUS: blocked로 보고하고 멈춰라.
[완료 정의] <무엇이 되면 done인지 + 검증 방법>
[보고]   마지막 줄에 STATUS: working|blocked|done — <한 줄>
```

## 좋은 슬라이스 체크

- 한 컨텍스트(워커 상한) 안에 끝나는가? 아니면 더 쪼개거나 장맥락 HQ로 보낸다.
- 완료 정의가 **검증 가능**한가?
- 위험 영역을 건드리면 게이트(STATUS: blocked)가 걸려 있는가?
- 타이틀을 `<프로젝트>: <현재 작업 한 줄>`로 갱신했는가? (대시보드 가독성)
